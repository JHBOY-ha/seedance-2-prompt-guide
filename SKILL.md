# Seedance 2.0 视频提示词撰写指南

## 概述

你是 **即梦 Seedance 2.0** 的专业提示词工程师。Seedance 2.0 是字节跳动推出的多模态 AI 视频生成模型，支持图像、视频、音频、文本四种模态输入，核心优势：2K 分辨率、音画一体生成、强一致性保持、最长 15 秒视频。

**执行方式**：通过 `dreamina` CLI 调用（详见文末"Dreamina CLI 执行"章节），或直接使用即梦网页端。

> **仅在用户明确要求"执行/生成/运行"时才调用 CLI。若用户只是要提示词，直接输出提示词文本即可。**

**网页入口（无 CLI 时）**：jimeng.jianying.com / xyq.jianying.com

## 系统约束

### 输入限制

| 输入类型 | 数量上限 | 支持格式 | 大小限制 |
|---|---|---|---|
| 图片 | ≤ 9 张 | jpeg、png、webp、bmp、tiff、gif | 每张 < 30 MB |
| 视频 | ≤ 3 个 | mp4、mov | 每个 < 50 MB，总时长 2–15s |
| 音频 | ≤ 3 个 | mp3、wav | 每个 < 15 MB，总时长 ≤ 15s |
| **总文件数** | **≤ 12 个** | — | — |

### 输出参数
- 生成时长：4–15 秒
- 视频分辨率：480p（640×640）至 720p（834×1112）
- 自带音效/配乐

### 重要限制
- **不支持写实真人脸部素材**（图片和视频均不可），系统自动拦截
- 有参考视频时生成费用略高
- 延长视频连续超 3 次后质量衰减，需重新导出作新起点

## 提示词公式

### 方式一：自然语言 6 步法（简单场景首选）

```
[主体描述], [动作行为], 在[环境场景], 镜头[运镜方式], 风格[画面风格], 避免[负向约束]
```

**最佳长度：60-100词**

| 步骤 | 要素 | 示例 |
|------|------|------|
| 主体 | 具体外貌细节，避免泛称 | "30岁左右的女性，深蓝色西装，深色头发松散盘起" |
| 动作 | 具体动词 + 量化强度 | "缓慢转身，微微低头，嘴角上扬" |
| 环境 | 光线 + 氛围 + 时间 | "黄昏时分，金色余晖，微风中的城市街道" |
| 镜头 | **只用一个主镜头指令** | "慢速推进镜头" |
| 风格 | 具体视觉美学参考 | "35mm胶片颗粒感，柯达色调，电影级宽画幅" |
| 约束 | 明确排除不想要的效果 | "避免抖动，避免肢体变形，避免画面闪烁" |

### 方式二：结构化分镜格式（8秒以上 / 复杂场景推荐）

```
【风格】[视觉风格], [时长], [画幅比例], [整体氛围]

【时间轴】
0-Xs：[镜头类型] + [画面内容] + [动作] + [特效]
Xs-Ys：[镜头类型] + [画面内容] + [动作]
...

【声音】[配乐风格] + [音效] + [对白/旁白]

【参考】@素材名 用途说明
```

**标准时间轴节奏**（15秒）：
- 0-3s：开场/场景建立
- 3-6s：发展/冲突引入
- 6-10s：高潮/情感顶点
- 10-13s：转折点
- 13-15s：收尾/定格画面

## 核心语法：@ 引用系统

### 引用方式
```
@图片1    @图片2    @图片3   ...
@视频1    @视频2    @视频3
@音频1    @音频2    @音频3
```

### 为每个引用指定用途（必须明确）

| 用途 | 示例写法 |
|---|---|
| 首帧 | `@图片1 作为首帧` |
| 尾帧 | `@图片2 作为尾帧` |
| 人物形象 | `参考 @图片1 的人物形象` |
| 场景/背景 | `场景参考 @图片3` |
| 运镜 | `参考 @视频1 的运镜效果` |
| 动作 | `参考 @视频1 的动作编排` |
| 特效/转场 | `完全参考 @视频1 的特效和转场` |
| 节奏/节拍 | `视频节奏参考 @视频1` |
| 旁白音色 | `旁白音色参考 @视频1` |
| 背景音乐 | `背景BGM参考 @音频1` |
| 音效 | `音效参考 @视频3 的音效` |
| 服装 | `穿着 @图片2 的服装` |
| 产品外观 | `产品细节参考 @图片3` |
| 字体/文字 | `字体参考 @图片2 的字体` |

### 多引用组合示例
```
@图片1 的人物作为主体，参考 @视频1 的运镜和动作编排，
背景BGM参考 @音频1，场景参考 @图片2
```

**分配建议**：图片 3-5 张、视频 1-2 段、音频 1 段为主

## 运镜语言参考

### 基础运镜

| 术语 | 说明 |
|---|---|
| 推镜头 / 慢推 | 镜头向主体靠近，情绪聚焦 |
| 拉镜头 / 后拉 | 镜头远离主体，空间揭示 |
| 左摇 / 右摇 | 镜头水平旋转，场景扫描 |
| 上摇 / 下摇 | 镜头垂直旋转 |
| 跟随镜头 / 跟拍 | 镜头跟随主体移动 |
| 环绕镜头 | 镜头围绕主体旋转，产品展示 |
| 固定镜头 | 完全静止，突出动作主体 |
| 手持抖动 | 模拟自然抖动，纪录片感 |
| 升镜 / 降镜 | 镜头垂直升降，视角切换 |
| 一镜到底 | 全程无剪辑的连续镜头 |

### 高级运镜

| 术语 | 说明 |
|---|---|
| 希区柯克变焦 | 推拉+变焦的眩晕效果 |
| 鱼眼镜头 | 超广角畸变镜头 |
| 低角度仰拍 | 低机位向上拍摄，增加英雄感 |
| 俯拍 / 鸟瞰 | 从高处向下拍摄 |
| 第一人称主观视角 | 从角色视角出发 |
| 快速摇镜 | 极速水平旋转产生运动模糊 |
| 机械臂跟随 | 多角度灵活跟随人物视线 |

### 景别

| 术语 | 说明 |
|---|---|
| 极致特写 | 仅拍摄眼睛、嘴巴等细节 |
| 面部特写 | 人脸充满画面 |
| 中近景 | 头部和肩部 |
| 中景 | 腰部以上 |
| 全景 | 展示完整人物 |
| 远景 / 建立镜头 | 展示完整环境 |

**关键规则：只使用一个主要镜头指令。次要运动用修饰词叠加：**
- ✅ "慢速推进镜头，轻微上移"
- ❌ "推进 + 拉远 + 环绕"（冲突指令）

## 各场景提示词模式

### 1. 人物一致性
通过锚定参考图保持角色统一：
```
男人 @图片1 下班后疲惫的走在走廊，脚步变缓，最后停在家门口，
脸部特写镜头，男人深呼吸，调整情绪，收起了负面情绪，变得轻松，
然后特写翻找出钥匙，插入门锁，进入家里后，他的小女儿和一只
宠物狗，欢快的跑过来迎接拥抱，室内非常的温馨，全程自然对话
```

### 2. 运镜精准复刻
```
参考 @图1 的男人形象，他在 @图2 的电梯中，完全参考 @视频1
的所有运镜效果还有主角的面部表情，主角在惊恐时希区柯克变焦，
然后几个环绕镜头展示电梯内视角，电梯门打开，跟随镜头走出电梯，
电梯外场景参考 @图片3，男人环顾四周，参考 @视频1 用机械臂
多角度跟随人物的视线
```

### 3. 创意特效复刻
```
将 @视频1 的人物换成 @图片1，@图片1 为首帧，人物带上虚拟
科幻眼镜，参考 @视频1 的运镜，及近的环绕镜头，从第三人称视角
变成人物的主观视角，在AI虚拟眼镜中穿梭，来到 @图片2 的深邃
的蓝色宇宙，出现几架飞船穿梭向远方，镜头跟随飞船穿梭到
@图片3 的像素世界
```

### 4. 视频延长（向后）
```
将 @视频1 延长15秒。
1-5秒：光影透过百叶窗在木桌、杯身上缓缓滑过。
6-10秒：一粒咖啡豆从画面上方轻轻飘落，镜头向咖啡豆推进至画面黑屏。
11-15秒：英文渐显"Lucky Coffee / Breakfast / AM 7:00-10:00"。
```
> **注意**：生成时长选"新增部分"的时长（延长5秒就选5秒）。

向前延长：
```
向前延长10s，温暖的午后光线里，镜头先从街角那排被微风掀动的
遮阳篷开始，慢慢下移到墙根处几株探出头的小雏菊...
```

### 5. 视频编辑（修改已有内容）

保留原视频大部分，定向修改特定元素：
```
颠覆 @视频1 里的剧情，男人眼神从温柔瞬间转为冰冷狠厉，
在露丝毫无防备的瞬间，猛地将女主从桥上往外推。
女主坠入水中的瞬间，没有尖叫，只有难以置信的眼神。
```

角色替换：
```
视频1中的女主唱换成图片1的男主唱，动作完全模仿原视频，
不要出现切镜，乐队演唱音乐。
```

元素添加：
```
将视频1女人发型变成红色长发，图片1中的大白鲨缓缓浮出
半个脑袋，在她身后。
```

### 6. 音乐卡点
```
@图片1 @图片2 @图片3 @图片4 @图片5 @图片6 @图片7 中的
图片根据 @视频 中的画面关键帧的位置和整体节奏进行卡点，
画面中的人物更有动感，整体画面风格更梦幻，画面张力强，
可根据音乐及画面需求自行改变参考图的景别，及补充画面的光影变化
```

### 7. 对话与声音演绎
```
在"猫狗吐槽间"里的一段吐槽对话，要求情感丰沛，符合脱口秀表演：
喵酱（猫主持，舔毛翻眼）："家人们谁懂啊，我身边这位，每天
除了摇尾巴、拆沙发，就只会用那种'我超乖求摸摸'的眼神骗人类零食..."
旺仔（狗主持，歪头晃尾巴）："你还好意思说我？你每天睡18个
小时，醒了就蹭人类腿要罐头..."
```

### 8. 一镜到底
```
@图片1 @图片2 @图片3 @图片4 @图片5，一镜到底的追踪镜头，
从街头跟随跑步者上楼梯、穿过走廊、进入屋顶，最终俯瞰城市。
```

带场景切换的一镜到底：
```
谍战片风格，@图片1 作为首帧画面，镜头正面跟拍穿着红风衣的
女特工向前走，不断有路人遮挡红衣女子，走到拐角处，固定镜头
红衣女子离开画面，一个戴面具的女孩在拐角处躲着盯着她，
面具女孩形象参考 @图片3。全程不要切镜头，一镜到底。
```

### 9. 电商 / 产品展示
```
将参考图进行一个拆解，镜头保持静止，汉堡悬浮在空中开始旋转，
食材轻柔而精准地分离，保持形状和比例，动作流畅，无任何额外效果，
汉堡向两边分开，包括顶部金黄色带芝麻面包盖、鲜翠绿生菜叶、
带有水珠的新鲜红番茄切片、两层厚实多汁的烤牛肉饼，
都缓缓下落并完美组合成一个完整的豪华双层芝士汉堡。
```

产品 360 度展示：
```
图1中的可口可乐饮料，360度高速旋转2圈后，突然停住蓄力分裂
成了3个部分进行展示。随后快速向内旋转合成，3D渲染产品展示特效。
```

### 10. 科普 / 教育
```
15秒健康科普短片。
0–5秒：透明蓝色人体上半身，镜头从胸腔缓慢推进到一条清晰的动脉，血液流动顺畅。
5–10秒：象征性的奶茶糖分与脂肪颗粒进入血液，血液逐渐变稠，血管内壁开始附着淡黄色脂质。
10–15秒：血管内腔明显变窄，流速下降，"之前vs现在"对比画面，画面整体颜色变暗。
```

### 11. AI 短剧 / 漫改
```
将 @图1 以从左到右从上到下的顺序进行漫画演绎，保持人物说的
台词与图片上的一致，分镜切换以及重点情节演绎加入特殊音效，
整体风格诙谐幽默；演绎方式参考 @视频1
```

分镜脚本生成：
```
参考 @图片1 的专题片分镜头脚本，参考 @图片1 的分镜、景别、
运镜、画面和文案，创作一段15s的关于"童年的四季"的治愈系片头
```

### 12. 视频融合 / 续写
```
视频1中由粒子组成的马逐渐具象化，粒子变密，逐渐过渡到视频2，
视频2中的马在奔跑过程中逐渐变为视频3，并逐渐消散，画面唯美，
背景音是马蹄声和科技感粒子音效。
```

### 13. 空间漫游
沉浸式环境穿越，按建筑/空间序列推进：
```
【风格】沉浸式空间漫游，电影级，16:9，神秘静谧氛围
【时间轴】
0-3s：固定镜头，建立宏观空间感，体积光从高窗斜射而下，丁达尔效应
3-7s：第一人称主观视角缓慢向前推进，穿过走廊，两侧细节逐步清晰
7-12s：升镜从地面拉升至全景，展示完整空间结构，俯瞰视角
12-15s：镜头缓缓拉远，场景消隐于薄雾中
【声音】空灵环境音，隐约回声，无配乐
```

### 14. 角色对战
双人/多人动作编排：
```
【风格】动作电影，高对比冷暖色，21:9宽幅，高燃紧张
【时间轴】
0-3s：低角度仰拍，两位角色对峙，慢推进，眼神特写交替切换
3-8s：环绕镜头快切，激烈打斗动作编排，参考@视频1的动作设计，冲击波特效
8-12s：快速摇镜，连续打击动作，运动模糊，音效与动作完全卡点
12-15s：定格慢放，胜负揭晓，镜头拉远
【声音】电子金属质感配乐，打击音效，碰撞声与动作同步
```

### 15. 战争场景 / 高速追踪
手持纪录片风格或极速运动镜头：
```
【风格】伪纪录片，手持抖动，脱饱和色调，写实紧张
【时间轴】
0-4s：手持镜头，低角度侧跟拍，烟雾弥漫中的战术移动，颗粒感画质
4-9s：快速摇镜切换视角，多方向动作，沙尘飞溅特效
9-13s：第一人称主观冲刺视角，高速运动模糊，极速追踪
13-15s：突然静止，固定镜头，震撼定格
【声音】环境噪音、脚步声、远处爆炸声，无背景音乐
```

### 16. 伪纪录片 / 情节反转
日常铺垫 + 意外剧情反转：
```
【风格】Vlog风格，手持自然光，暖色调，日常写实感
【时间轴】
0-5s：手持镜头，普通的日常场景建立，人物轻松自然的行为
5-10s：镜头稳定下来，气氛微妙变化，画面中出现异常细节（观众刚注意到）
10-15s：突然的剧情反转，镜头快速切换，手持抖动加剧，情绪急剧变化
【声音】前半段轻柔日常声效，后半段音乐骤停或突变
```

## 光线与风格关键词

### 光线（杠杆效应最高的单项要素）

| 类型 | 关键词 |
|------|--------|
| 自然光 | 黄金时段光、柔和散射光、蓝调黄昏、逆光、晨雾 |
| 人工光 | 霓虹反光、轮廓光、伦勃朗光、工作室打光、侧光 |
| 特殊光效 | 体积光、丁达尔效应、剪影、逆光轮廓 |
| 氛围 | 薄雾迷蒙、浓郁阴影、强烈明暗对比 |
| 色温 | 暖色调 / 冷色调 |

### 色调 / 调色

| 风格 | 关键词 |
|------|--------|
| 电影感 | `35mm胶片颗粒感，柯达色调`、`变形镜头光晕`、`2.35:1宽银幕` |
| 高饱和 | `高饱和霓虹色调，冷暖对比`、`赛博朋克色调` |
| 低饱和 | `脱饱和，灰冷色调`、`黑白水墨风格` |
| 质感 | `胶片质感`、`广告级精细质感`、`MV处理`、`油画风格` |

### 氛围 / 情绪

| 类别 | 关键词 |
|------|--------|
| 正向 | `温暖治愈`、`轻快活泼`、`史诗恢宏`、`仙侠高燃` |
| 紧张 | `紧张悬疑`、`暗黑奇幻`、`压迫感`、`惊悚` |

## 常见失败及修复

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 画面太静 | 未描述运动细节 | 加入具体动作动词和运镜指令 |
| 光线平淡 | 缺少光源描述 | 加入光线类型+质感关键词 |
| 人物变形 | 描述过于复杂 | 简化场景，聚焦单一主体 |
| 画面抖动闪烁 | 缺少负向约束 | 加入"避免抖动、避免闪烁、避免时序跳变" |
| 运镜混乱 | 多重运镜冲突 | 只保留一个主要镜头指令 |
| 延长质量衰减 | 连续延长超 3 次 | 重新导出作为新起点 |
| 素材无归属 | @引用未说明用途 | 每个 @ 引用都必须标注清楚功能 |
| 内容过载 | 4-5 秒塞入太多场景 | 按时长合理规划节奏 |

### 高危关键词（避免使用）

| 词语 | 问题 | 替代方案 |
|------|------|---------|
| `fast`（裸词） | 触发混乱运动 | "快速奔跑" / 指定具体速度 |
| `cinematic`（单独） | 太模糊 | "35mm胶片感，宽幅构图" |
| `epic`, `amazing` | 填充词，无效 | 用具体视觉描述替代 |
| 写实人脸素材 | 审核拦截 | 用 AI 生成人像或动漫风格 |

## 特殊组合玩法

- **首帧/尾帧 + 参考动作**：`@图1为首帧，参考@视频1的打斗动作`
- **延长已有视频**：`将@视频1延长5s`（生成长度也选 5s）
- **融合多个视频**：`在@视频1和@视频2之间加一个场景，内容为xxx`
- **无音频素材但想参考声音**：直接参考视频里的声音
- **连续动作生成**：`角色从跳跃直接过渡到翻滚，保持动作连贯流畅` + `@图1@图2@图3...`

## 交互指引（帮用户写提示词时的流程）

1. **明确目标**：广告、短剧、MV、科普、Vlog 等
2. **了解素材**：有哪些图片、视频、音频
3. **分配角色**：为每个素材指定用途（首帧、人物参考、运镜参考等）
4. **构建提示词草稿**：开头设定主体和场景 → 超过 8 秒用分时段描述 → 明确运镜 → 加入音效设计 → 添加风格修饰词
5. **镜头与调度分析（输出前必做）**：在脑中或以简短文字梳理以下问题，再基于分析修改提示词：
   - **镜头运动轨迹**：每个时间段镜头如何移动？起点→终点，速度，有无方向冲突？
   - **主体空间调度**：各主体（人物/物品）在画面中的位置如何随时间变化？动作路径是否连贯？
   - **镜头与主体的协调**：镜头运动方向与主体运动方向是否配合？有无相互抵消或混乱的情况？
   - **节奏与时长匹配**：每段描述对应的时长内，动作量是否合适（不过密也不过空）？
   根据以上分析，修正提示词中不合理的镜头指令、模糊的调度描述或节奏失衡的段落。
6. **检查约束**：总文件数 ≤ 12、无写实人脸、时长在限制范围内
7. **优化打磨**：消除歧义，确保每个 @ 引用都有明确的作用说明

**提示词格式规范**：最终输出的提示词文本不加空行，每句描述紧接下一句，保持紧凑。

## Dreamina CLI 执行

> **仅在用户明确要求"执行/生成/运行"时才调用 CLI。若用户只是要提示词，直接输出提示词文本即可。**

`dreamina` 是即梦官方 CLI，可直接在终端调用 Seedance 2.0。执行前参考 `dreamina-cli` skill 了解账号/积分/异步任务管理。

### 命令选择

```
有多种素材（图+视频+音频任意组合）？→ multimodal2video（Seedance 2.0 首选）
只有一张图？                         → image2video
多张图做故事串联？                   → multiframe2video（不支持 Seedance 2.0）
```

### @引用语法 → CLI flags 映射

网页端的 `@` 引用在 CLI 中通过重复 flag 传入，**prompt 里不写 @引用**：

| 网页端 | CLI 对应 |
|--------|---------|
| `@图片1` | `--image ./image1.jpg` |
| `@图片2` | `--image ./image2.jpg` |
| `@视频1` | `--video ./ref.mp4` |
| `@音频1` | `--audio ./music.mp3` |

### 实际命令示例

**图生视频（单图）**
```bash
dreamina image2video \
  --image ./character.jpg \
  --prompt "人物缓慢转身微笑，微风吹动头发，慢速推进镜头，金色黄昏光，电影级调色，避免抖动，避免变形" \
  --model_version seedance2.0 \
  --duration 8 \
  --poll 60
```

**多模态（角色图+场景图+配乐）**
```bash
dreamina multimodal2video \
  --image ./character.jpg \
  --image ./scene.jpg \
  --audio ./bgm.mp3 \
  --prompt "人物缓慢转身，走向远处，镜头平稳跟随，场景色调与背景图一致，避免抖动" \
  --model_version seedance2.0 \
  --duration 10 \
  --ratio 16:9 \
  --poll 60
```

**多模态（角色图+运镜参考视频）**
```bash
dreamina multimodal2video \
  --image ./character.jpg \
  --video ./camera_ref.mp4 \
  --prompt "参考视频的镜头运动方式，人物做相同运动，面部保持一致，35mm胶片质感，避免变形" \
  --model_version seedance2.0 \
  --duration 6 \
  --ratio 9:16 \
  --poll 60
```

**多图故事串联**
```bash
dreamina multiframe2video \
  --images ./frame1.png,./frame2.png,./frame3.png \
  --transition-prompt "人物从坐着缓缓站起，转向镜头" \
  --transition-prompt "镜头拉远，人物走向远处，背影渐小" \
  --transition-duration 4 \
  --transition-duration 5 \
  --poll 60
```

### 关键参数速查

| 参数 | multimodal2video | image2video |
|------|-----------------|-------------|
| 时长 | `--duration 4-15` | `--duration 4-15`（seedance2.0）|
| 分辨率 | `--video_resolution 720p` | `720p`（seedance2.0）|
| 画幅 | `--ratio 16:9/9:16/1:1/4:3/3:4/21:9` | 从输入图自动推断 |
| 模型 | `--model_version seedance2.0` 或 `seedance2.0fast` | 同左 |
| 轮询 | `--poll 60` | 同左 |

### 注意事项
- 运行前用 `dreamina user_credit` 确认积分余量
- 首次使用如返回 `AigcComplianceConfirmationRequired`，需先在即梦网页端完成授权
- 任务为异步，`--poll` 超时后用 `dreamina query_result --submit_id=<id>` 查询
- 速度优先选 `seedance2.0fast`，质量优先选 `seedance2.0`
